## 迁移计划

### 1. 新版 `exec` 工具预期行为

#### 1.1 接口

```
exec({
  command?: string,   // 要执行的命令；查询后台任务时可省略
  directory?: string, // 工作目录，可选
  server?: string,    // SSH 远程主机，可选
  task_id?: string,   // 后台任务标识（如 "task_1"），用于查询结果
})
```

#### 1.2 执行新命令（提供 `command`）

1. **会话与连接准备**
   - 根据 sessionID 确定 socket 路径 `/tmp/yse-tmux/yse-{sid}.sock`。
   - 若 `server` 指定，建立/复用 SSH 多路复用连接（ControlMaster），保证远程连接持久。
   - 确保 tmux 会话存在；首次创建时，包含一个窗口名为 `main` 的窗格。
   - 所有窗格设置 `remain-on-exit`，以便保留输出。

2. **主窗格空闲保证**
   - 因为工具调用为同步阻塞，`main` 窗格在接收新命令时必定空闲（shell 处于提示符等待状态），无需额外检查。

3. **命令注入**
   - 生成唯一标记 `PS1=__YSE_{uuid}__`。
   - 若 `directory` 提供，先发送 `cd` 命令。
   - 通过 `tmux send-keys` 向 `main` 窗格发送设置 PS1 及目标命令，结尾追加 `Enter`。

4. **同步等待（最长 2 分钟）**
   - 轮询 `main` 窗格输出（每 200ms 捕获一次），检测 PS1 标记是否出现。
   - **标记出现**：
     - 命令已完成。截取命令行之后、标记之前的完整输出。
     - 返回 `{ status: "completed", output: "<...>" }`。
   - **2 分钟超时**：
     - **主窗格降级**：
       1. 将当前 `main` 窗格重命名为 `task-N`（N 为全局自增序号）。
       2. 创建一个新的 `main` 窗格，启动 shell，处于空闲状态。
     - 生成不透明任务标识 `task_id = "task_N"`。
     - 返回 `{ status: "running", task_id: "task_N", message: "命令仍在执行中，稍后使用 task_id 获取结果。" }`。
   - **AbortSignal 触发**：
     - 立即执行与超时相同的降级流程，将主窗格降级为后台任务，新建 `main`。
     - 抛出 `AbortError`，并在错误对象中附带 `task_id`，供智能体捕获后查询。

5. **后台任务窗格**
   - `task-N` 窗格内同样已注入 PS1 标记，用于后续精确捕获输出。
   - 窗格保留 `remain-on-exit`，命令结束后不自动销毁。

#### 1.3 查询后台任务（提供 `task_id`）

1. 根据 `task_id` 定位对应窗格（如 `task-1`）。若不存在，返回错误 `"任务不存在或已被清理"`。
2. 轮询该窗格输出，检测 PS1 标记（等待最长 2 分钟）：
   - 标记出现：提取输出，返回 `{ status: "completed", output: "<...>" }`。
   - 再次超时：返回 `running` 状态和相同 `task_id`，不销毁任务。
   - `AbortSignal` 触发：抛出 `AbortError`，任务保持后台运行，后续可继续使用同一 `task_id` 查询。
3. 任务完成后，窗格保留，供人类查看或由回收机制清理（如闲置 1 小时后关闭）。

#### 1.4 无参数调用

- 返回所有可供查询的长时间任务

---

### 2. SSH 连接持久化

- 首次使用 `server` 时，工具建立 SSH 多路复用连接：`ssh -M -S /tmp/yse-ssh-{hash} -f -N {server}`。
- 后续所有远程 tmux 操作通过 `-o ControlPath=...` 复用该连接，无重复认证。
- 连接与会话 ID 绑定，在智能体会话结束时清理。

---

### 3. 错误处理与边界情况

| 场景 | 行为 |
|------|------|
| 查询的 `task_id` 不存在 | 返回错误 `"任务不存在或已被清理"` |
| 后台任务窗格被外部关闭 | 返回错误 `"任务窗格丢失，无法获取结果"` |
| 远程服务器断开 | 尝试重建 SSH 连接；失败则返回连接错误 |
| 同时指定 `command` 和 `task_id` | 返回错误 `"不能同时指定 command 和 task_id"` |

---

### 4. 迁移步骤

1. **实现替换**：用新版 `exec` 替换旧版，保持 `command`、`directory`、`server` 兼容；增加可选 `task_id`。
2. **旧会话处理**：不需要兼容

---

### 5. 与人类工作流的对应关系

- 工具始终阻塞等待短命令，与普通 bash 体验一致。
- 超时命令被自动放入“后台”（降级为 `task-N`），智能体可继续执行新命令（新 `main` 窗格），就像人类按下 `Ctrl+Z` 后打开一个新终端。
- 查询后台任务相当于人类执行 `fg` 或查看后台任务输出。
- 人类通过 `tmux attach -S /tmp/yse-tmux/yse-{sid}.sock` 进入，看到 `main` 和若干 `task-N` 窗格，历史输出完整可审计。
