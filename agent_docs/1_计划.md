# 盐水鹅 项目计划

## 定位
基于邮件中转的分布式聊天 + 插件自动化系统，支持桌面与移动端。

## 架构

```
yse/                          # Cargo workspace
├── core/                     # yse-core: 纯业务逻辑，零平台依赖
├── desktop/                  # yse-desktop: Tauri 桌面端胶水
├── mobile/                   # yse-mobile: Tauri 移动端胶水
└── frontend/                 # Vue + TDesign UI
```

**core** 是所有平台共享的 library，各 Tauri 二进制只做胶水。

## 通讯模型

### 邮件层
```
From: 随机伪装用户@热门邮箱.com    (gmail/outlook/yahoo/qq/163/proton/icloud)
To:   user@qq.com                (用户注册的真实邮箱)
Subject: (空)

Content-Type: multipart/mixed

[Part 1: text/plain, 加密的消息 JSON]
  [12B Nonce][ChaCha20-Poly1305(JSON)]

[Part 2-N: application/octet-stream, 加密的文件附件]
  Content-Disposition: attachment; filename="f1.bin"
  [12B Nonce][ChaCha20-Poly1305(file)]
```

### 消息 JSON（解密后）
```json
{
  "protocol": "yse.v1",
  "from": "alice@yse.org",
  "to": "bob@yse.org",
  "timestamp": 1719876543210,
  "id": "uuid-v4",
  "text": "看看这个",
  "files": [
    {"name": "方案.pdf", "mime": "application/pdf", "size": 1048576, "enc_name": "f1.bin"}
  ],
  "meta": {}
}
```

### 加密
- 密钥：用户密码 → Argon2id → 32 字节对称密钥
- 算法：ChaCha20-Poly1305，12字节随机 Nonce
- 每个文件独立加密，不同的 Nonce

## 数据流

```
[用户A 发送]
  → 前端 invoke "send"
  → core 构造 JSON + 加密 + disguise(from)
  → SMTP 发送到用户A邮箱
  → 用户A邮箱服务器存储

[所有客户端拉取]
  → IMAP 收件箱全量扫
  → 尝试用共享密钥解密
  → 解密失败 → 跳过（非 yse 邮件）
  → 解密成功 → 去重 (id) → 按 timestamp 排序 → 入 pending 队列
  → 按 from/to 查本地映射表 → 分发到对应插件进程
  → 若 to==自己的虚拟地址 → 同时推送到前端聊天界面
```

## 模块划分

### core 模块

| 模块 | 功能 | 关键依赖 |
|------|------|----------|
| `crypto` | Argon2id 密钥派生 + ChaCha20-Poly1305 加解密 | `argon2`, `chacha20poly1305` |
| `message` | 消息模型定义 + JSON 序列化 + MIME 组装/解析 | `serde`, `mailparse` |
| `disguise` | 随机伪装发件人生成 | (无) |
| `store` | Storage trait + SqliteStorage 实现 | `rusqlite` |
| `email/imap` | IMAP 轮询 (IDLE + 10s fallback) | `async-imap` |
| `email/smtp` | SMTP 发送 (multipart mixed) | `lettre` |
| `plugin` | PluginManager: 进程生命周期 + stdio JSON-RPC | (子进程) |
| `router` | 消息按虚拟地址分发到插件/UI | — |
| `config` | 配置持久化 (邮箱凭据、密钥、插件映射表) | — |
| `event` | 内部 EventBus | `tokio::sync::broadcast` |

### desktop/mobile 模块
- Tauri 命令封装 (`commands.rs`)
- 平台特定代码（系统托盘、文件选择、通知等）

### 前端模块
- ChatView — 聊天界面，显示收发的消息
- ConfigView — 邮箱配置、密钥设置
- PluginView — 插件管理（安装/启停/映射规则）
- LogView — 运行日志

## 实现顺序

| 步 | 内容 | 验证 |
|----|------|------|
| 1 | Workspace 初始化 + 空 crate 结构 | `cargo build` 全 workspace 通过 |
| 2 | `crypto` — argon2 派生 + chacha20 加解密 | 单元测试加解密一致性 |
| 3 | `message` — 消息模型 + JSON 序列化 | 单元测试序列化/反序列化 |
| 4 | `disguise` — 随机发件人生成 | 单元测试输出格式 |
| 5 | `store` — Storage trait + SqliteStorage | 单元测试 CRUD |
| 6 | `email` — IMAP 拉取 + SMTP 发送 + MIME 解析 | 连接真实邮箱测试 |
| 7 | `plugin` — 进程管理 + JSON-RPC | 测试插件通信 |
| 8 | `router` — 消息分发 + 有序交付 | 集成测试 |
| 9 | desktop + mobile Tauri 胶水 | `cargo tauri dev` 可用 |
| 10 | 前端 Vue + TDesign | 人工交互 |
